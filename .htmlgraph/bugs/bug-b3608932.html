finalize-yaml ignores meta.track_id — always creates a fresh track

Done Medium Priority

Part Of:

  • trk-8cf41009

Description

htmlgraph plan finalize-yaml always creates a new track regardless of whether meta.track_id is set in the YAML. This orphans any pre-existing track the plan was associated with. Behavior should be: if meta.track_id is set and the track exists, reuse it; if not, create a new one. Currently forces users to delete the duplicate track after finalize.
