## Problem When sdk.features.start('feat-X') is called, UserQuery backfill correctly stamps feat-X. But subsequent PreToolUse hooks attribute tool events to a DIFFERENT in-progress feature (feat-Y). ## Evidence (2026-03-10, v0.33.57) Called sdk.features.start('feat-74b2cf48'). DB shows: - UserQuery events: feat-74b2cf48 (correct) - All tool events: feat-5922d683 (WRONG) ## Root Cause PreToolUse hook determines active feature independently from SDK start(). It queries features table for most recently updated in-progress feature. With multiple features in-progress, picks wrong one. ## Key Files - src/python/htmlgraph/hooks/pretooluse.py (how feature_id determined for tool events) - src/python/htmlgraph/hooks/context.py (get_active_feature or similar) - src/python/htmlgraph/sessions/features.py (start_feature + _backfill_turn1_userquery) ## Fix Direction PreToolUse hook should query for the feature most recently started via sdk.features.start() in the CURRENT SESSION, not just any in-progress feature. Could use primary_feature marker in session record or last-started timestamp.
