Description

Clicking a session node in the graph opens the transcript view but shows 'No messages in this session' with 0 messages, 0 tool calls, no model, no duration. These are sessions recorded by SessionStart hook but never actually used (no PostToolUse events). They should be filtered out of the graph — only include sessions with meaningful activity. The current filter requires >5 events via agent_events JOIN, but the filter is on feature_id != '' which excludes sessions with no feature attribution. Need stricter filter: require at least 1 message AND at least 1 tool call in the tool_calls/messages tables, not just agent_events.
