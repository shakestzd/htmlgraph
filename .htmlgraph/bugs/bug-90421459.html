rewrite-yaml strips slice approved fields — approvals out of sync with YAML

Done Medium Priority

Part Of:

  • trk-8cf41009

Description

htmlgraph plan rewrite-yaml discards the approved field on slices and design, even when the input YAML has approved: true. This creates confusion: the YAML shows approved: false but the actual approval state lives in SQLite plan_feedback. Either: (1) rewrite-yaml should preserve the approved fields from the input YAML, or (2) the approved field should be removed from the YAML schema entirely since approvals are stored in SQLite.
