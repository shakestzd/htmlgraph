Description

Yolo mode creates a git worktree which copies .htmlgraph/ from the commit point. Work items created after that commit are invisible. Tracking events recorded in the worktree are lost when it's cleaned up. The fix: .htmlgraph/ should always resolve to main's copy, not the worktree's. All CLI commands and hooks should read/write to the main project's .htmlgraph/ regardless of CWD. This means the --project-dir flag or CLAUDE_PROJECT_DIR env var should point to main, not the worktree.
