Feedback API vs finalize-yaml section key mismatch — underscore vs hyphen

Implemented In:

Description

POST /api/plans/{id}/feedback accepts section names like slice_1, slice_2 (underscore) and writes them to plan_feedback. But finalize-yaml (plan_finalize_yaml.go:120) queries for slice-1, slice-2 (hyphen). Result: approvals submitted via the API are invisible to finalize-yaml, which then skips all slices as not approved. Fix: standardize on one separator. The dashboard UI should match whatever finalize-yaml reads.
