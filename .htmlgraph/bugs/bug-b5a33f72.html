Orchestrator and researcher agent do not prioritize documentation search in debugging workflows

Todo Medium Priority

Caused By:

  • feat-f4106ebf

Part Of:

  • trk-3ab54383

Description

When debugging third-party library issues (e.g. marimo mo.ui.chat), the orchestrator delegates source code reading and guesswork instead of requiring documentation search first. The researcher agent prompts also lack 'search official docs first' as a mandatory step. This led to 20+ minutes wasted reading marimo Python source when the official docs would have answered the question immediately. Fix: (1) Orchestrator system prompt must enforce docs-first debugging order: official docs → changelog → GitHub issues → then source code. (2) Researcher agent definition must include web search of official documentation as step 1 for any third-party library investigation.
