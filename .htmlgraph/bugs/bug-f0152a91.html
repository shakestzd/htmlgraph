batch apply bypasses hierarchy enforcement — creates orphan features

Done Medium Priority

Part Of:

  • trk-f2a1a880

Caused By:

  • spk-865a44af

Description

executeBatchApply calls Features.Create() directly without warnMissingFields. YAML specs with no track block silently create orphan features that the next orphan purge will destroy. Fix: add validation in executeBatchApply that errors if spec.Features is non-empty and spec.Track.Title is empty.
