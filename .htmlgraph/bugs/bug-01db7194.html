Description

When a git worktree is created, it snapshots plugin/hooks/bin/htmlgraph at creation time. Yolo dev mode launches with --plugin-dir plugin/, so hooks resolve to the WORKTREE copy, not the global ~/.local/bin/htmlgraph. Any CLI fixes made after worktree creation are invisible to yolo agents. Fix options: (1) yolo init copies the current binary into the worktree, (2) hooks resolve to the global binary instead of plugin-dir binary, (3) symlink worktree binary to parent.
