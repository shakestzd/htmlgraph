Description

Command: `htmlgraph yolo --track trk-6e10c73c`. Expected: attach to the existing worktree/branch for the track. Actual: fails with `git worktree add failed: exit status 255` because the underlying `git worktree add` is invoked with `new branch 'trk-6e10c73c'` (implicit -b), and the branch already exists from a prior run. Verbatim output: ``` htmlgraph yolo --track trk-6e10c73c Update available: v0.55.6-58-g75f8b760 → v0.55.6. Run: htmlgraph plugin install git worktree add failed: exit status 255 Preparing worktree (new branch 'trk-6e10c73c') fatal: a branch named 'trk-6e10c73c' already exists ``` Likely fix site: whatever invokes `git worktree add` in yolo's track-mode setup. Needs to detect an existing branch/worktree for the track and attach to it (`git worktree add ` without `-b`, or `cd` into the existing worktree) instead of unconditionally creating a new branch. Also surfaces a UX point — the update-available banner prints on every invocation even while the run is failing.
