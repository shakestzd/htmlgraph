Description

When clicking a work item from the graph view, navigating to its detail, then clicking Back to Board, the work view shows an empty state (Work 0) instead of the kanban board. Root cause: closeWorkDetail shows the board container but features[] is empty because fetchFeatures was never called (user came from graph, not work view). Fix: closeWorkDetail should call fetchFeatures when features.length === 0.
