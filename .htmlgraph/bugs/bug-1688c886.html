Work item status not auto-completed when implementation is committed

Todo Medium Priority

Caused By:

  • Auto-complete work items when implementation is committed

Part Of:

  • Work Item Traceability — hierarchy enforcement, file attribution, smart linking

Description

When a feature's implementation is committed in a worktree (code done, tests passing), the work item status remains 'todo'. The developer must manually run 'htmlgraph feature complete feat-xxx' for each item. This led to trk-f8fd5c66 showing 1/8 done when 7/8 were actually implemented. The system should auto-detect completion — either via post-commit hooks that match commit messages to feature IDs (e.g., feat-xxx in commit trailer), or via the YOLO/worktree workflow that runs 'complete' after quality gates pass. At minimum, 'htmlgraph review' or 'htmlgraph check' should flag features with committed code but todo status.
