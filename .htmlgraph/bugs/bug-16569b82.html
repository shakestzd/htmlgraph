sanitizePathID truncation collision on long file paths

Done Medium Priority

Caused By:

  • spk-865a44af

Part Of:

  • trk-f2a1a880

Description

sanitizePathID in reindex_files.go truncates to 32 chars with no hash suffix. Two file paths sharing the same first 32 sanitized characters produce the same composite primary key, causing UpsertFeatureFile to silently overwrite. Fix: append 8-char sha256 hash suffix after truncation.
