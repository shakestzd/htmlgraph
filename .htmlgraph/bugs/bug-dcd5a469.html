Guard accepts completed features as active — feature complete does not clear session active_feature_id

Done Medium Priority

Caused By:

  • feat-032e6a56

Part Of:

  • trk-97f85b3b

Implemented In:

  • session 559faa0b-464a-43ca-95a6-65097d374f34

Description

When htmlgraph feature complete runs, the feature's status changes to done but the session's active_feature_id column still points to it. The work item guard then checks ctx.FeatureID != '' and allows edits, even though the feature is no longer active. Two fixes needed: (1) feature complete should clear sessions.active_feature_id for sessions where it matches, and (2) the guard should join to features table and verify status = 'in-progress' not just that the column is non-empty.
