Description

Agents need to update plan YAML files during review iteration (modify slices after critique, clear critique for fresh review, update design after feedback). Currently no CLI command supports this — agents must bypass the write guard. Add plan rewrite-yaml that: (1) reads existing plan, (2) accepts modified YAML from stdin or --file, (3) validates against planyaml schema, (4) atomically replaces the file. This ensures all .htmlgraph/plans/*.yaml files have valid structure. The write guard should remain blocking — all plan writes go through validated CLI commands.
