## Problem When SSE auto-refresh fires (~10 seconds via WebSocket), expanded parent rows in the activity feed collapse back to closed state. Users lose their place. ## Root Cause saveExpandState()/restoreExpandState() saves expanded event_ids to a JS Set, then restores after htmx morph. Fails because: 1. DOM structure changes when new events arrive (new rows inserted, event counts change) 2. restoreExpandState() fires via setTimeout(150ms) but morphed DOM may not be settled 3. State keys may not match new DOM elements after server re-render ## Reproduction 1. Open dashboard, click parent row to expand child events 2. Wait 10-15 seconds for SSE auto-refresh (or run a Bash command) 3. Expanded section collapses back ## Key Files - src/python/htmlgraph/api/templates/partials/activity-feed-hierarchical.html (saveExpandState/restoreExpandState JS + htmx morph handlers) - src/python/htmlgraph/api/templates/dashboard-redesign.html (refreshActivityFeed() + SSE handler) ## Suggested Fix Option A: Move expand state to CSS classes that survive idiomorph merge Option B: Use htmx:beforeMorph to save + htmx:afterSettle to restore with data-event-id matching Option C: Don't re-render expanded sections, only append new events to existing expanded DOM ## Verified Failing v0.33.57, 2026-03-10.
