Description

In YOLO mode CLAUDE_ENV_FILE is unset so HTMLGRAPH_SESSION_ID is never exported. bug start resolves session via .active-session file (stale), PreToolUse hook uses event.session_id from CloudEvent. These differ, so bug start writes active_feature_id to wrong session row. Guard blocks despite bug being in-progress. Fix: add fallback in checkYoloWorkItemGuard to check work item status directly when sessionHasLinkedFeature fails.
