Smart work item placement — auto-resolve track and parent feature on create

Todo Medium Priority

Part Of:

  • trk-f2a1a880

Description

When creating a bug or feature, the system should automatically determine the correct track and parent feature based on semantic content matching — not just accept the --track flag blindly or auto-link to an arbitrary feature. Current behavior: auto-linker picks the wrong feature (e.g., bug about work item completion linked to session ingester). Expected behavior: analyze the bug/feature title and description, search existing tracks and features for semantic relevance, suggest or auto-assign the correct parent. This could be: (1) a CLI pre-create step that proposes placement, (2) orchestrator/agent rules that validate placement before delegation, or (3) a post-create 'htmlgraph lint' that flags misplaced items.
