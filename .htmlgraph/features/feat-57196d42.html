On the JS side, add a chat panel to internal/plantmpl/templates/plan_page.gohtml: input textarea, send button, scrollable bubble list (user/assistant differentiated by data-role attr), streaming text append via EventSource, auto-scroll. Load chat history via GET /api/plans/{id}/feedback on page open (filter action="chat_message" rows). Graceful degradation: hide panel when /api/plans/{id}/chat returns 503 (Claude unavailable).
