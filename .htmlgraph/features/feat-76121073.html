Lazy session registration — CLI commands self-heal missing session records

Todo Medium Priority

Part Of:

  • trk-06a29a51

Description

Every CLI command that runs inside a Claude Code session should ensure the session is registered in the DB. Resolution: (1) check HTMLGRAPH_SESSION_ID env var, (2) fallback to most recently modified .jsonl in ~/.claude/projects//, (3) if session not in DB or marked completed, upsert as active. This removes the single-point-of-failure dependency on the session-start hook.
