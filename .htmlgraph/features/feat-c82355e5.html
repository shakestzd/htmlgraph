Guards apply to all sessions, not just YOLO mode

Done Medium Priority

Part Of:

  • trk-97f85b3b

Description

Currently work item attribution, research-before-writing, test-before-commit, diff-review, code health, and file budget guards only fire when ctx.IsYoloMode is true. This creates a two-tier system where interactive sessions skip the discipline. Fix: make most guards always-on (interactive + YOLO), with only worktree branch requirement, subagent grace period, and steps validation as YOLO-only. Some guards can be warnings in interactive mode, blocks in YOLO mode.
