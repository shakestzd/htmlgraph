Create `cmd/htmlgraph/projects.go` with a `projectsCmd()` function returning a `*cobra.Command` for `htmlgraph projects`. Add two subcommands: `htmlgraph projects list` — calls `registry.Load(registry.DefaultPath())`, iterates entries, prints a table with columns: NAME, DIR, LAST_SEEN, STATUS (exists/missing), ITEMS. STATUS is "exists" if `filepath.Join(entry.ProjectDir, ".htmlgraph")` is stat-able, "missing" otherwise. ITEMS count is derived by opening the project DB read-only via `registry.OpenReadOnly(filepath.Join(entry.ProjectDir, ".htmlgraph", "htmlgraph.db"))` and querying feature + bug + spike counts — skip silently on open error. Do NOT use `workitem.Open(entry.ProjectDir, ...)` here as that runs schema migrations. `htmlgraph projects prune` — calls `reg.Prune()`, prints pruned dirs, calls `reg.Save()`. Register `projectsCmd()` in `main.go` via `rootCmd.AddCommand(projectsCmd())`. Add "projects" to the PersistentPreRunE skip list so the subcommand works without a local `.htmlgraph/` directory.
