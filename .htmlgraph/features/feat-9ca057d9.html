Description

Extend `cmd/htmlgraph/dashboard/js/app.js` to detect global mode by calling `GET /api/mode` on startup (not `/api/projects`, which returns an HTML 404 in single-project mode due to the file server fallback). If the response has `mode == "global"` the app enters global mode. In global mode, render a project switcher panel above the existing nav — a `` or pill-list of project names plus an "All Projects" option. Selecting a specific project re-scopes all subsequent API calls using the `?project=` query parameter on existing routes (e.g., `/api/sessions?project=`, `/api/features?project=`). Selecting "All Projects" calls `/api/projects/all/stats` for the stats panel and renders a merged session list from all projects. Add a `buildProjectUrl(projectId, endpoint)` helper in `helpers.js` that returns `/api/${endpoint}?project=${projectId}` when `projectId` is set or `/api/${endpoint}` for the single-project fallback. Single-project mode (when `/api/mode` returns `{"mode":"single"}`) must behave exactly as today — no regressions. No new JS dependencies — vanilla JS only, matching the existing codebase style.
