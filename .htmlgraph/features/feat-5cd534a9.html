Description

In `cmd/htmlgraph/serve.go`, add a `--global` bool flag to `serveCmd()`. When `--global` is set, call a new `runGlobalServer(port int)` function instead of `runServer`. `runGlobalServer` loads the registry, iterates entries where status==exists, opens each project DB read-only via `registry.OpenReadOnly(filepath.Join(entry.ProjectDir, ".htmlgraph", "htmlgraph.db"))` — never via `workitem.Open` which runs migrations. Collect `[]*sql.DB` mapped by stable project ID (`entry.ID`, the 8-char SHA256 prefix stored in the registry entry). Add new API routes to the mux: `GET /api/projects` — returns `[]ProjectSummary` JSON where each summary contains `{id, name, dir, lastSeen, featureCount, bugCount, spikeCount}` using the stable `entry.ID` as the project ID. `GET /api/mode` — returns `{"mode":"global","projects":[...]}` in global mode; this endpoint is also added to the single-project server returning `{"mode":"single"}`. Extend existing routes with a `?project=` query parameter: `GET /api/sessions?project=` dispatches to `sessionsHandler` for the matched project's `*sql.DB`; `GET /api/features?project=` dispatches to `featuresHandler`; `GET /api/stats?project=` dispatches to `statsHandler`. `GET /api/projects/all/stats` — aggregates stat counts across all projects. When `?project` is absent or the route is hit without global context, fall through to single-project behaviour. Reuse existing `corsMiddleware` and handler functions — pass `*sql.DB` and `htmlgraphDir` as explicit arguments rather than closures where needed. All cross-project DB connections are read-only. Existing `runServer` (single-project) must remain unchanged.
