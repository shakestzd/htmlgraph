Description

In `cmd/htmlgraph/main.go`, extend the existing `rootCmd.PersistentPreRunE` to call `registry.Load(registry.DefaultPath())`, then `reg.Upsert(projectDir, name, gitRemoteURL)`, then `reg.Save()` after the existing `agent.EnsureSession` call. Derive `projectDir` from the already-resolved `filepath.Dir(hgDir)`. Derive `name` as `filepath.Base(projectDir)`. Derive `gitRemoteURL` via `paths.GetGitRemoteURL(projectDir)` — but ONLY when the existing registry entry for `projectDir` has an empty `GitRemoteURL`; if the entry already has a URL populated, skip the `git remote get-url origin` subprocess entirely and pass the cached value to `Upsert`. This ensures the git subprocess runs at most once per project, not on every CLI invocation. Skip the registry upsert (return `nil`) when `findHtmlgraphDir()` returns an error (same graceful-degradation pattern already used for EnsureSession). The entire upsert + save must complete within 10 ms on an SSD for the common case (cached remote URL, no git subprocess); the first-invocation case may take up to 30 ms due to git process startup.
