Description

The finalize button should trigger a complete pipeline, not just YAML status update + HTML export. Steps: (1) Apply accepted amendments via rewrite-yaml. (2) Increment meta.version. (3) Create track + features from approved slices via htmlgraph plan finalize-yaml — this generates real work items. (4) Embed the generated track ID and feature IDs back into the YAML and notebook cells so the exported HTML links to them. (5) Export static HTML with all interactive widgets replaced by static text — the archival record includes chat history, track/feature links, and version number. The version number is critical because work items continue evolving after the plan is finalized — the version tracks which plan state generated them.
