TrackSpecPlan

Beads Convergence: Graph Scheduling + Compaction + Coordination

Planned High Priority Created: 2026-03-13

Achieve all benefits of Beads (dependency graph, semantic compaction, multi-agent coordination, context management) while preserving HtmlGraph strengths (real-time observability, event system, dashboard, hooks, HTML format)

Overview

Port key Beads capabilities into HtmlGraph architecture: dependency-aware scheduling, atomic claim, CIGS scaling, semantic compaction, wisps, and rich relationship types.

Context

HtmlGraph has ~60% of infrastructure already (graph engine, archival manager, WIP limits) but none of it receives data. Beads uses Dolt+SQL for dependency graph and scheduling. HtmlGraph can achieve the same with SQLite + existing graph.py code.

Requirements

⏳ SQLite-backed CIGS queries replacing HTML file scanning

must-have

⏳ SDK blocked_by() edge writing + ready queue CLI

must-have

⏳ Atomic claim primitive for multi-agent coordination

must-have

⏳ Tiered semantic compaction with Claude Haiku

should-have

⏳ Ephemeral wisp signals between agents

should-have

⏳ Rich relationship types beyond blocked_by

nice-to-have

Acceptance Criteria

  1. ⏳ CIGS guidance generated from SQLite in <50ms (not file I/O) test_cigs_sqlite_performance
  2. ⏳ htmlgraph ready lists unblocked items test_ready_queue
  3. ⏳ sdk.features.claim() is atomic compare-and-swap test_atomic_claim
  4. ⏳ 30-day items compacted to ~30% original size test_tier1_compaction
  5. ⏳ Wisps expire after TTL and appear in CIGS test_wisp_lifecycle
  6. ⏳ blocked_by edges traversed by graph engine test_dependency_traversal
