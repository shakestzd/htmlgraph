Production Reliability Hardening

Planned High Priority Created: 2026-03-07

FastAPI lifespan migration, SQLite PRAGMA centralization, observability stack (OTel/Prometheus/Sentry/correlation IDs), reliability testing, and durable orchestration

Overview

Harden HtmlGraph for production by normalizing infrastructure (FastAPI/SQLite), adding a full observability stack, introducing property-based reliability testing, and evaluating durable workflow orchestration for long-running agent coordination.

Context

HtmlGraph has grown beyond the HTML-only/minimal-deps constraint. The stack now includes FastAPI, SQLite WAL, Pydantic, and Redis fallback logic. With that foundation, production-grade reliability and observability are high-ROI additions that reduce toil and surface bugs earlier.

Acceptance Criteria

  1. ⏳ FastAPI starts cleanly with lifespan, no deprecation warnings test_lifespan_startup()
  2. ⏳ All SQLite connections use identical PRAGMA set from single source test_pragma_consistency()
  3. ⏳ PRAGMA optimize runs on schedule without manual intervention test_pragma_optimize_scheduled()
  4. ⏳ Pydantic strict mode rejects invalid ingress payloads test_strict_validation()
  5. ⏳ OTel spans visible in trace output for all API routes test_otel_spans()
  6. ⏳ /metrics endpoint returns Prometheus counters after requests test_prometheus_metrics()
  7. ⏳ Request IDs propagate through all log lines for a single request test_correlation_id_propagation()
  8. ⏳ schemathesis finds 0 new API violations from schema test_schemathesis_clean()
