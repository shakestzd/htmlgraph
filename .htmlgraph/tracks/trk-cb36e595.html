Description

Re-architect htmlgraph serve --global from shared-process-with-query-param dispatch to child-process-with-reverse-proxy isolation. Addresses the core architectural flaw surfaced during MVP smoke testing (2026-04-09): the current global server opens read-only DB handles to every project and dispatches project-scoped requests via ?project= — which conflates read aggregation with project-scoped mutation and makes cross-project contamination possible by construction. The hardened design runs one child htmlgraph-serve process per active project on a dynamic port, with the global server serving only aggregate reads and reverse-proxying /p//* to the matching child. OS-level process isolation, correct-by-construction env/CWD/hook context, crash isolation, smaller routing attack surface, composes correctly with the (session_id, agent_id) attribution model. Also solves parallel attribution collisions as a side-effect because each project gets its own sessions table writer queue.
