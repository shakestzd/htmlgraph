## Problem The dashboard auto-refreshes the activity feed every 30 seconds. When the refresh fires, any expanded rows (conversation turns, nested child events) snap back to collapsed/default state, losing the user's view. ## Exact Mechanism ### 1. Polling trigger (line 4-7, activity-feed-hierarchical.html) The outer container div declares HTMX polling:
Every 30 seconds, HTMX GETs /views/activity-feed and swaps the entire outer container using idiomorph (morph:outerHTML). It replaces the outerHTML of .activity-feed-view — the entire subtree — with server-rendered HTML. ### 2. Morph extension is present but insufficient - idiomorph-ext.min.js IS loaded (static/idiomorph-ext.min.js, line 8 of dashboard-redesign.html) - hx-ext="morph" and hx-swap="morph:outerHTML" are set on the container Idiomorph is a DOM-morphing library that attempts to match old and new DOM nodes by identity and update them in place, preserving state. HOWEVER, it only preserves state automatically for: - Input field values - Focus - Scroll position (on the morphed element itself) It does NOT automatically preserve: - CSS class state (e.g. collapsed class removed = expanded row) - data-children-loaded="true" attributes set by JavaScript at runtime - The expand-icon.expanded CSS class (set by toggleChildren/handleParentClick) ### 3. Why morph fails here The expanded/collapsed state is encoded purely as CSS classes on DOM elements: - .turn-children.collapsed — hidden when collapsed (display:none), no collapsed = expanded - .expand-icon.expanded — rotates the triangle 90 degrees - .expand-toggle-turn.expanded — same for conversation-turn toggles - .child-children.collapsed — nested depth collapsed state - data-children-loaded="true" — set by JS to mark lazy-load already done Idiomorph matches nodes by id/key. The parent row IDs are stable (event UUIDs via id="{{ group.parent.event_id }}"). However, the server-rendered HTML always delivers the child containers with class="turn-children collapsed" and no expanded classes. When idiomorph morphs the old expanded DOM to the new server DOM, it REMOVES the runtime-added classes (expanded, etc.) because morphing applies the server's attribute values, which never include "expanded". Additionally, data-children-loaded is always "false" in the server HTML (line 37 of activity-feed-hierarchical.html): data-children-loaded="false" So even if a user had clicked to lazy-load children, after a morph that attribute is reset to false and the expand icon state is lost. ### 4.
