Researcher: Investigated Go binary .htmlgraph/ path resolution for worktree bug (bug-c45eef1a). Key findings: (1) Central function ResolveProjectDir() is in packages/go/internal/hooks/runner.go:93. It checks CLAUDE_PROJECT_DIR env var first, then event.CWD, then process CWD, then walks up from CWD. (2) findHtmlgraphDir() in packages/go/cmd/htmlgraph/main.go:90 is the CLI equivalent — checks --project-dir flag, then CLAUDE_PROJECT_DIR env var, then walks up from os.Getwd(). (3) CLAUDE_PROJECT_DIR is written to CLAUDE_ENV_FILE in writeEnvVars() at packages/go/internal/hooks/session_start.go:108. This is a chicken-and-egg: session-start MUST resolve projectDir BEFORE it can write CLAUDE_PROJECT_DIR — so the first session-start call has no env var set. (4) The walk-up logic in ResolveProjectDir (runner.go:111-122) finds .htmlgraph/ by walking up from cwd. In a worktree (e.g., .claude/worktrees/feat-79d10093/) the walk-up will find the main repo .htmlgraph/ because it walks up from the worktree path through .claude/worktrees/feat-79d10093/ -> .claude/worktrees/ -> .claude/ -> main-repo/ which DOES have .htmlgraph/. HOWEVER the CWD check at runner.go:99-103 only checks if .htmlgraph exists directly in cwd — it does not walk up. So if event.CWD is the worktree path, the CWD direct check fails but walk-up succeeds. (5) No git worktree detection (git rev-parse --git-common-dir or git worktree list) exists anywhere in path resolution. (6) The issue title says CLAUDE_ENV_FILE unset — if CLAUDE_ENV_FILE is not set, writeEnvVars() silently returns at session_start.go:93, meaning CLAUDE_PROJECT_DIR never gets propagated. Without it, all subsequent hook invocations must rely on walk-up. (7) Walk-up from worktree path DOES resolve correctly to main .htmlgraph/ (6 levels up: feat-id -> worktrees -> .claude -> main-repo), so the core path resolution actually works IF the walk-up runs. Root cause: When CLAUDE_ENV_FILE is unset, writeEnvVars is a no-op, so CLAUDE_PROJECT_DIR is never set. But walk-up should still find .htmlgraph/. The real issue may be that event.CWD = worktree path, direct stat check fails, process CWD also = worktree path, direct stat check fails, then walk-up runs and should succeed (10 iterations is enough for 6 levels). Recommended approach to fix: add git-common-dir detection in ResolveProjectDir as a step between CWD check and walk-up — run 'git -C cwd rev-parse --git-common-dir' and derive the main repo root from that.
