## Research: Latest Claude Code CLI Features (2025-2026) ## Date: 2026-03-05 ## Sources: code.claude.com/docs, GitHub CHANGELOG, platform.claude.com/docs/agent-sdk --- ## 1. HOOK EVENTS - COMPLETE CURRENT LIST (17 total) HtmlGraph currently handles: UserPromptSubmit, Stop, SessionStart, SessionEnd, PreToolUse, PostToolUse, PostToolUseFailure, SubagentStop, TeammateIdle, TaskCompleted = 10 events MISSING from HtmlGraph hooks.json (7 events not wired): - SubagentStart (fires when subagent spawns - subagent-start.py exists but NOT in hooks.json) - PreCompact (fires before context compaction - matcher: manual|auto) - PermissionRequest (fires when permission dialog appears) - InstructionsLoaded (fires when CLAUDE.md or .claude/rules/*.md loaded into context) - ConfigChange (fires when config file changes - matcher: user_settings|project_settings|local_settings|policy_settings|skills) - WorktreeCreate (fires when worktree created via --worktree or isolation:worktree) - WorktreeRemove (fires when worktree removed) COMPLETE EVENT TABLE (official 2026 docs): | SessionStart | startup|resume|clear|compact matcher | | UserPromptSubmit | no matcher support | | PreToolUse | tool name matcher | | PermissionRequest | tool name matcher (NEW - NOT IN HTMLGRAPH) | | PostToolUse | tool name matcher | | PostToolUseFailure | tool name matcher | | Notification | permission_prompt|idle_prompt|auth_success|elicitation_dialog | | SubagentStart | agent type name (WIRED IN PYTHON BUT NOT hooks.json) | | SubagentStop | agent type name | | Stop | no matcher | | TeammateIdle | no matcher | | TaskCompleted | no matcher | | InstructionsLoaded | no matcher (NEW - NOT IN HTMLGRAPH) | | ConfigChange | user_settings|project_settings|local_settings|policy_settings|skills (NEW) | | WorktreeCreate | no matcher (NEW - NOT IN HTMLGRAPH) | | WorktreeRemove | no matcher (NEW - NOT IN HTMLGRAPH) | | PreCompact | manual|auto matcher (NEW - NOT IN HTMLGRAPH) | | SessionEnd | clear|logout|prompt_input_exit|bypass_permissions_disabled|other | --- ## 2. NEW HOOK INPUT FIELDS (Critical for HtmlGraph) All hooks now receive agent_id and agent_type when inside a subagent: agent_id - Unique identifier for the subagent (NEW) agent_type - Agent name e.g. "Explore", "security-reviewer" (NEW) This is EXACTLY what we have been working around with our SUBAGENT_SUFFIXES hack! The official API now provides agent_id and agent_type directly in hook input. HtmlGraph should use these fields instead of parsing session_id suffixes. SessionStart matcher values: startup|resume|clear|compact SessionEnd matcher values: clear|logout|prompt_input_exit|bypass_permissions_disabled|other SubagentStop now includes: - agent_id - agent_type - agent_transcript_path (path to subagent JSONL transcript) - stop_hook_active - last_assistant_message (final response text) Stop hook now includes: - last_assistant_message (final response text) PreCompact includes: - transcript_path (full path before compaction summary) - session_id, cwd, permission_mode, hook_event_name --- ## 3. NEW HOOK HANDLER TYPES (Beyond just command) HtmlGraph only uses type:"command" hooks. New types available: - type:"http" - POST JSON to URL, get JSON response (can block/allow/modify) - url, headers (with env var interpolation), allowedEnvVars, timeout - Error handling: non-2xx = non-blocking (unlike exit 2 for commands) - type:"prompt" - Send event data to Claude model for yes/no decision - prompt field with ARGUMENTS placeholder, model field - type:"agent" - Spawn a subagent to verify conditions before deciding - prompt, model, timeout (default 60s) HTTP hooks enable HtmlGraph to POST events to a local web server for real-time dashboard updates WITHOUT needing file-based IPC. Hook handler common fields (NEW): - statusMessage: Custom spinner text while hook runs - once: true = run only once per session (Skills only) --- ## 4. OPENTELEMETRY - FULL OBSERVABILITY API (NEW - NOT USED BY HTMLGRAPH) Enable: CLAUDE_CODE_ENABLE_TELEMETRY=1 METRICS exported: claude_code.session.count - session starts claude_code.lines_of_code.count - code modified (type: added|removed) claude_code.pull_request.count - PRs created claude_code.commit.count - commits created claude_code.cost.usage - cost in USD (model attribute) claude_code.token.usage - tokens (type: input|output|cacheRead|cacheCreation, model) claude_code.code_edit_tool.decision - accept/reject per tool claude_code.active_time.total - active time seconds (type: user|cli) EVENTS exported (via OTel logs): claude_code.user_prompt - prompt_length, prompt (opt-in with OTEL_LOG_USER_PROMPTS), prompt.id claude_code.tool_result - tool_name, success, duration_ms, error, decision_type, decision_source, tool_result_size_bytes, tool_parameters (bash_command, git_commit_id for successful git commits) claude_code.api_request - model, cost_usd, duration_ms, input_tokens, output_tokens, cache_read_tokens, cache_creation_tokens, speed (fast|normal) claude_code.api_error - model, error, status_code, duration_ms, attempt, speed claude_code.tool_decision - tool_name, decision (accept|reject), source CORRELATION: prompt.id UUID ties all events to a single user prompt. Standard attributes on all: session.id, app.version, organization.id, user.account_uuid, user.id, user.email, terminal.type Exporters: otlp (grpc/http), prometheus, console Dynamic headers: otelHeadersHelper script in settings.json HtmlGraph opportunity: Subscribe to OTel events via OTLP receiver to get cost data, token counts, and model info WITHOUT needing hook-based tracking. OTel provides exact cost_usd per API call and model name - currently estimated in HtmlGraph. --- ## 5. SUBAGENT SYSTEM - NEW CAPABILITIES New frontmatter fields for agent definitions: - background: true - always run as background task - isolation: worktree - run in temporary git worktree (auto-cleanup if no changes) - memory: user|project|local - persistent memory across sessions - skills: [list] - preload skill content into subagent context at startup - mcpServers: inline or by name - hooks: inline definitions scoped to subagent lifetime - maxTurns: N CLI flag --agents JSON - define subagents for single session without files. Subagent transcript storage: ~/.claude/projects/{project}/{sessionId}/subagents/agent-{agentId}.jsonl Subagents can be RESUMED across sessions using agent ID. Task tool renamed to Agent tool in v2.1.63 (Task alias still works). Agent(agent_type) syntax in tools field restricts spawnable subagents. HtmlGraph constants.py has SUBAGENT_SUFFIXES hack for detecting subagent sessions. With agent_id + agent_type in hook inputs, this workaround is no longer needed. --- ## 6.
