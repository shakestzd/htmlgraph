## Context - 2,231 tests collected across 116 test files in tests/python/ - Suite runs ~83 seconds wall-clock with uv run pytest - pytest-xdist is already installed (pyproject.toml dev dep) but NOT enabled in addopts - pytest.ini already has: testpaths, --strict-markers, slow/integration markers defined - autouse fixture cleanup_env_vars runs on every single test (harmless, ~0 overhead) - conftest.py already has pytest_collection_modifyitems wiring @pytest.mark.serial -> xdist_group ## Sleep Hotspots Found (concrete time waste) 1. test_orchestration.py:142 — time.sleep(2) in test_get_results_by_task_id_polling_behavior - Waits for a background thread to create a delayed spike - Real fix: reduce poll_interval param or parametrize with smaller delays in tests - Mark @pytest.mark.slow 2. test_cross_agent_attribution.py:16 — time.sleep(1.1) in test_cross_agent_attribution - Comment says "Avoid ID collision" — IDs are likely timestamp-based - Real fix: inject a frozen/mock clock so IDs are deterministic without sleeping - Quick fix: reduce to time.sleep(0.01) if IDs only need ordering not 1s gaps - Mark @pytest.mark.slow or fix the ID generation 3. test_cost_monitor.py:358 — time.sleep(0.1) repeated in loop - Fix: mock time.time() so you control timestamps without real sleeping 4. test_session_registry.py:272,304,555,616 — multiple time.sleep(0.001) and 0.01 - Tiny, but autouse fixture means 2231 × overhead still adds up - Fix: mock datetime or use monkeypatch 5. test_orchestration.py poll_interval=1 — total polling adds ~2-10s for that test ## Biggest Wins (ranked by expected impact) ### 1.
