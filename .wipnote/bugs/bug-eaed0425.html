Description

Each release run bumps the canonical plugin.json version but never invokes wipnote plugin build-ports, so the codex-marketplace and gemini-extension version stamps lag the source-of-truth by one release. Hit three times in a single session; caught post-hoc by the session-exit reconciler, which means the release tag itself points at trees with the lag (only main gets the catch-up commit after the fact). Functionally fine for the GoReleaser binary install path, but consumers fetching ports by tag see the drift. Fix: invoke wipnote plugin build-ports before the version-bump commit so the release commit contains coherent ports across all targets.
