Description

First-time-use surfacing: in a non-wipnote project (e.g. user's actual app), 'wipnote check --gate' fails with 'load gate allowlist: open /plugin/config/quality-gate-flake-allowlist.json: no such file or directory'. This file only exists in the wipnote repo. Downstream effect: feature/bug/spike completion requires a passing gate record, but the gate can never produce one in any other project. The entire completion flow is broken for real users. CRITICAL Show-HN blocker — affects every cloner/installer doing actual work. Fix direction: make the allowlist optional (treat missing as empty), or ship a sensible default at a wipnote-managed user-local path, or relax the gate to not require this file at all.
