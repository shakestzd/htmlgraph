Description

Reported via GH issue #113. Finalizing a plan via the dashboard auto-promotes approved slices into features; running wipnote plan promote-slice from CLI for the same slices creates duplicates. Plan YAML feature_id only records the latest promote call, orphaning the dashboard-created feature. 9 slices became 18 features. Promote-slice should be idempotent — short-circuit if slice already promoted, returning existing id. Need optional --force/--re-promote flag.
