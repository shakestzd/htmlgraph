Canonical completion blocked by Git index writability

Todo Medium Priority

Part Of:

  • trk-08dcbb33

Description

Completion currently depends on Git autocommit succeeding after writing .wipnote artifacts. If Git cannot create .git/index.lock, such as a read-only worktree or locked Git common dir, wipnote reopens the item even though canonical state could be durable. Make canonical .wipnote state persist atomically first, record Git sync as committed or pending, add retry via wipnote sync, and keep strict blocking only as configurable dogfood/dev policy.
