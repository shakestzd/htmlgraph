Description

Reproduction (v0.60.4 darwin/arm64): after 'wipnote upgrade' lays down ~/.local/share/wipnote/codex-marketplace, 'wipnote codex' prints 'WIPNOTE AGENTS NOT LOADED' with Codex CLI error 'invalid marketplace file ...codex-marketplace: marketplace root does not contain a supported manifest'. Root cause: prior fix for bug-53e78bcc (commit 46b94966f) updated the validator isValidHarnessTree to accept both flat and deep codex layouts so resolveSharedTreePath succeeds, but did not fix the registration call. cmd/wipnote/codex.go:772 passes the tree ROOT to 'codex plugin marketplace add', while the actual marketplace.json lives at /.agents/plugins/marketplace.json. Codex CLI looks for the manifest at the path it was given and errors. Affects all dogfooders running wipnote codex from a v0.60.3+ install. Likely fix: pass /.agents/plugins/marketplace.json (or its parent dir) to Codex; or restructure bundled tree so marketplace.json lives at root.
