Relates To:

Description

Companion to the Write-guard scope bug. When a spike touches /tmp/plan.yaml as a planning scratch buffer, completion refuses with 'code-bearing (touched 1 source path(s) outside .wipnote/, e.g. /tmp/plan.yaml) but has zero linked source commits.' The path-tracker should not classify /tmp (or any path outside the project root) as a source path that requires a commit link. Even with --accepted-advisory, the path remains in the 'touched source paths' set and the gate-record requirement kicks in next. Fix direction: same scope-to-project-root as the Write guard — paths outside the project root are not project source. Planning-only spikes that never touched real project code should complete cleanly.
