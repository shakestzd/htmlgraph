install.sh incorrectly hard-errors on linux_arm64 though the asset is published

Todo Medium Priority

Part Of:

  • trk-0677c709

Description

scripts/install.sh has an explicit linux_arm64 unsupported-platform gate that hard-errors with a build-from-source hint. The gate was added on the (wrong) assumption that no linux_arm64 asset was published. In fact v0.60.1 (and v0.60.0) DO publish wipnote_VERSION_linux_arm64.tar.gz via the existing GoReleaser matrix. Affects: AWS Graviton, Asahi Linux, Raspberry Pi 4/5, devcontainers on Apple Silicon hosts. Fix: remove the linux_arm64-specific gate from install.sh, and add linux_arm64 to the supported-assets line in README.md and docs/index.md. End-to-end smoke test on linux_arm64 is possible directly in this devcontainer.
