Description

scripts/install.sh extracts the GoReleaser tarball but moves only the 'wipnote' binary to BIN_DIR; the bundled plugin/, codex-marketplace/, and gemini-extension/ directories are left in the temp dir and cleaned up by trap. Result: 'wipnote claude' (and codex/gemini equivalents) fails with 'wipnote plugin tree not found'. Founder hit this on macOS Apple Silicon immediately after the documented install. The Homebrew formula (.goreleaser.yml brews block) correctly installs all three trees to share/wipnote — the curl-install path must mirror this layout. Fix: extend install.sh to copy plugin/, codex-marketplace/, gemini-extension/ from the extracted tarball into a WIPNOTE_SHARE_DIR (default $HOME/.local/share/wipnote/), with env override and clean replace on upgrade. Smoke-testable in this linux_arm64 devcontainer (verify share-tree layout after install).
