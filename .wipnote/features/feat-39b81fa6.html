Description

New CLI subcommand `wipnote migrate normalize-paths` that walks `.wipnote/` and rewrites absolute paths to repo-relative in place. Safety preconditions (must pass before any write): a. Working tree clean — run `git -C status --porcelain .wipnote/`; abort with a clear error if dirty. Use `--allow-dirty` to bypass (records audit reason). b. Collision check — for `feature_files` (which has `UNIQUE(feature_id, file_path)`), pre-scan: if two rows for the same feature would normalize to the same relative path, list them in the dry-run output. Default behavior is to MERGE (take earliest `first_seen`, sum sessions); `--no-merge-collisions` causes the migration to abort instead. Targets (7, corrected from initial 6): 1. `agent_events.tool_input` — SINGLE-encoded JSON; unmarshal once, walk path keys, re-marshal. (Initial plan said doubly-encoded; that was wrong.) 2. `feature_files.file_path` (SQLite TEXT) — with collision detection. 3. `agent_events.input_summary` free-text — regex-replace WorktreePath embeds. 4. `pending_subagent_starts.cwd` 5. `sessions.project_dir` (SQLite) — newly added; missed in initial design. 6. ` ` in `.wipnote/sessions/*.html` 7. `affected_files` properties in `.wipnote/{features,bugs,spikes}/*.html` Out-of-scope (documented explicitly): - `sessions.transcript_path` — Claude's private machine-local store. - `.wipnote/.active-session` — transient per-session JSON; rewritten on next SessionStart anyway. Idempotency: re-running on already-relative records is a no-op. Achieved by skipping any value that doesn't match `HostPathPattern`. Backup + dry-run + restore: - `--backup` (default on): copies touched HTML to `.wipnote/.backup-/`. - `--dry-run`: prints proposed changes and pending collisions; no writes. - Companion: `wipnote migrate restore-paths --from .wipnote/.backup-/` copies backups back into place. Print summary: N files rewritten, M values normalized, K skipped as already-relative, J marked unresolved, P collisions merged, A collisions aborted (if --no-merge-collisions). Rendering audit: confirm `wipnote feature show`, `wipnote blame`, `wipnote lineage`, and the dashboard template render relative paths correctly (resolve against current repo root for display, strip `unresolved:` prefix). Update each call site as needed.
