Description

Add `NormalizeToRepoRelative(absPath, repoRoot string) (relPath string, ok bool)` to `internal/paths/resolve.go`. It: 1. If `absPath` is already relative, return `(absPath, true)` unchanged. 2. If `repoRoot == ""`, resolve canonically: a. Try `ResolveViaGitCommonDir(filepath.Dir(absPath))`. b. If that returns `.git` (main worktree case) or empty, walk up from `filepath.Dir(absPath)` looking for the nearest `.wipnote/` directory. c. If both fail and `absPath` matches `HostPathPattern`, return `("unresolved:" + absPath, true)`. Otherwise return `(absPath, true)` (treat as already-relative-but-exotic). 3. Apply `filepath.EvalSymlinks` to both `repoRoot` and `absPath` before relativization. 4. Compute `filepath.Rel(canonicalRoot, evaledAbsPath)`; if the result starts with `..`, return `("unresolved:" + absPath, true)`. 5. Convert the relative path to forward slashes via `filepath.ToSlash` and return `(rel, true)`. Add a process-scoped cache (`sync.Map` keyed by `filepath.Dir(absPath)`) so `git rev-parse` is invoked at most once per unique directory per process lifetime — critical because slice-2 wires this into hot-path hooks (PreToolUse/PostToolUse fire on every tool call). Promote the `hostPathPattern` regex out of `cmd/wipnote/` into `internal/paths/hostpattern.go` so both the precommit gate and the runtime normalizer reference the same definition. Add a sibling `MustNormalize(absPath, repoRoot string) string` that returns the normalized path on success or the original `absPath` on failure, logging at debug level. Use this in hot paths where the hook shouldn't fail on a normalization edge case.
