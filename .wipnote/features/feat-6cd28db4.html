Also add it to scripts/deploy-all.sh and CI so stale generated trees can neither be committed nor shipped.

# Spec: Pre-commit wipnote plugin check-ports drift gate

## Problem
Mechanically eliminates the "forgot build-ports after a generator edit"
class on every harness with zero agent reliance (the feat-e78221c7
failure mode).

## Decisions
**Trigger:** a commit/deploy touching internal/pluginbuild,
packages/plugin-core, or plugin/ without regenerating the target trees.
**State:** durable filesystem - full regenerate into a tempdir, then
recursive diff against committed trees; no checksum/manifest cache.
**Payload:** non-zero exit listing each drifted path; pre-commit
invocation scoped to staged generator/manifest/plugin paths; also wired
into scripts/deploy-all.sh and CI.
**Rejected:** checksum-manifest approach (rejected - the checksum itself
drifts and adds a second source of truth); unscoped pre-commit regen on
every commit (rejected - adds 5-30s latency to unrelated commits; scope
to staged paths); per-harness bespoke drift checks (rejected - drifts,
defeats parity; the diff must run from the single shared build-ports
generator).
